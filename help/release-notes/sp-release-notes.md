## AEM 6.4.8.0 に含まれる機能 {#what-s-included-in-aem}

AEM 6.4.8.0 は、新機能、お客様からリクエストされた主な機能強化、パフォーマンス、安定性、セキュリティの改善が含まれる重要なアップデートです。これは、AEM 6.4 の一般リリース以降にリリースされた機能です。 **2018 年 4 月。**

また、これは累積的なもので、 6.4.8.0 には、その前にリリースされたすべてのAEM 6.4 サービスパックが含まれます。

このサービスパックリリースの主な特徴は次のとおりです。

* 組み込み型のリポジトリ（Apache Jackrabbit Oak）をバージョン 1.8.20 に更新しました。

* WCM-RTE の日本の Web サイトでは、ワードラップ機能がサポートされています。

* 日本語の Web サイトでは、改行および改行の処理がサポートされています。

* 日本語の文や行を適切な位置で区切る BUNSET メソッドの使用をサポートするようになりました。

* Correspondence Management の CCR UI で、ドイツ語ロケールの 10 進数値がサポートされるようになりました。

* SOAP Web サービスを使用したフォームデータモデルの統合で、要素の選択グループまたは属性がサポートされるようになりました。

* AEM Assetsは、 [!DNL Adobe I/O].

* ContextHub にバンドルされている jQuery のバージョンを 3.2.1 に更新しました。

## 変更について {#list-of-changes}

### Sites {#sites}

* AEM Sitesページの URL にコロンまたはパーセンテージ記号が含まれている場合、基になるブラウザーが応答を停止し、CPU サイクルにスパイクが表示される (NPR-32368、NPR-31917)。
* AEM Sitesページを編集用に開き、コンポーネントをコピーした場合、一部のプレースホルダーでは貼り付け操作が使用できない。(NPR-32328)
* アクティベーションワークフローのリクエストに参照されているアセットが含まれていない (NPR-32304)。
* ブループリントを作成すると、レコード数が 80 を超える場合は、最初の 80 件のレコードのみが表示されます。 残りのレコードはブループリントで空白行が表示されます（NPR-32058）。
* ユーザーが、必須フィールドに情報を提供せずにコンテンツフラグメントを保存できる（NPR-31988）。
* コアエクスペリエンスフラグメントコンポーネントで設定されたパスで自動ナビゲーションが機能しない (NPR-31921)。
* リッチテキストエディター (RTE) でテーブルセルのタイプを変更すると、次のエラーが発生します。
   `Error: No common ancestor found, cannot continue` (NPR-31916)。
* コンテンツが同じフォルダー内に移動されると、ページ移動オプションが無効になる (NPR-31841)。
* 日本語の文章を BUNSETST 法で分割し、改行を適切な位置に分割するサポートを追加しました。(NPR-31836)
* リッチテキストエディター (RTE) でハイパーリンクを編集した場合、新しく選択されたパスは保存されない (NPR-31659)。
* マルチフィールドコンポーネントを削除し、削除を取り消すと、コンポーネントは復元されますが、データは復元されません (NPR-31617)。

#### AEM 6.4.7.0 {#experience-manager-6470}

AEM 6.4.7.0 は、パフォーマンス、安定性、セキュリティおよび主なお客様向けの修正と、AEM 6.4 の一般リリース以降にリリースされた機能強化を含む重要なアップデートです。 **2018 年 4 月。**

また、これは累積的なもので、 6.4.7.0 には、その前のすべてのAEM 6.4 サービスパックリリースが含まれています。

AEM 6.4.7.0 の主な特徴は次のとおりです。

* 組み込み型のリポジトリ（Apache Jackrabbit Oak）をバージョン 1.8.17 に更新しました。
* サイトページの設定バージョンを削除する際のサポートを追加しました。
* 並べ替え可能な、作成日の新しい列が **DAM リスト** アセット検索結果の表示とオン **リスト** 表示 (NPR-31311)。
* 次に基づくアセットの並べ替え **名前** 次の場所で列が許可されています： **リスト** 表示
* 再処理およびバッチアップロードのバッチサイズとワークフローステップのタイムアウトが、Dynamic Mediaの UI から設定できるようになりました。
* この `pdfBrochure` は、IPS でメモリを保存するために、Scene7 クラウド設定で false に設定されています。

#### AEM 6.4.4.0 {#experience-manager-6440}

AEM 6.4.4.0 は、パフォーマンス、安定性、セキュリティおよび主なお客様向けの修正と、AEM 6.4 の一般リリース以降にリリースされた機能強化を含む重要なアップデートです。 **2018 年 4 月。**

AEM 6.4.4.0 の主な特徴は次のとおりです。

* 組み込み型のリポジトリ（Apache Jackrabbit Oak）をバージョン 1.8.11 に更新しました。
* サービスバージョンの HTTP リクエストが頻繁に発生するのを避けるために、サービスバージョンのキャッシュのサポートが追加されました。
* 自動タグの削除のサポートを追加しました。
* カタログウィザードに無限スクロールを実装しました。
* コミュニティサイトに応じて権限を制限する移植機能。
* Sling Granite コンテンツヘルスチェックのパフォーマンスを修正（キャッシュ、非同期実行、除外リスト）しました。
* assetpicker ボタンが pagethumbnail ダイアログに追加されました。
* フィールド上でのツールチップの配置を許可する新しいプロパティが追加されました。
* Multifield 内での ColorPicker のサポートを改善しました。
* コンテンツフラグメントの clientlibs の数値入力マルチフィールドで、空の値を無視するチェックを追加しました。
* Microsoft Translator Text API v3 のサポートを有効にしました。

#### AEM 6.4.3.0 {#experience-manager-6430}

AEM 6.4.3.0 は、パフォーマンス、安定性、セキュリティおよび主なお客様向けの修正と、2018 年 4 月のAEM 6.4 の一般リリース以降にリリースされた機能強化を含む重要なアップデートです。

また、これは累積的なもので、 6.4.3.0 には、その前のすべてのAEM 6.4 サービスパックリリースが含まれています。

AEM 6.4.3.0 の主な特徴は次のとおりです。

* 組み込み型のリポジトリ（Apache Jackrabbit Oak）をバージョン 1.8.9 に更新しました。
* アダプティブフォームテンプレートでの allowedPaths プロパティのサポートが追加されました。
* AEMでのパネルベースのアセット検索の強化
* ログインページのクロスサイトスクリプティング (XSS) を修正しました。
* UI 計測機能が改善されました。
* FormData の処理を改善しました。
* Multifield 内の項目の命名処理を改善しました。
* 選択時のプレースホルダー項目（カード表示およびリスト表示）の処理を改善しました。
* Managed ServicesのAdobe IMS認証とAdmin Consoleサポートを追加しました。

#### AEM 6.4.1.0 {#experience-manager-6410}

AEM 6.4.1.0 は、パフォーマンス、安定性、セキュリティおよび主なお客様向けの修正と、2018 年 4 月のAEM 6.4 の一般リリース以降にリリースされた機能強化を含む重要なアップデートです。

AEM 6.4.1.0 はAEM 6.4 GA 上にインストールできます。 このサービスパックの主な特徴は次のとおりです。

* 組み込み型のリポジトリ（Apache Jackrabbit Oak）をバージョン 1.8.3 に更新しました。
* 拡張スマートタグが導入されました。
* デザインピッカー、タグピッカーの修正（ソース VM とターゲット VM を自動に変更）
* 値を文字列として保存するための typeHint のサポートを追加しました。
* Mail Service に SMTP over TLS のサポートが追加されました。
* DMS7 での HTTP フォワーダのプロキシ処理の修正。
* /etc/clientlibs/social/thirdparty/handlebars/source/handlebars.jsバージョン 1.3 に更新しました。
* DMHybrid オプトアウトおよびオプトインパッケージを修正しました。
* スマート切り抜きの修正。
* OOTB 設定値を新しい場所（ /etc から/conf）に移行しました。
* 配信サーバーの顧客設定にカラープロファイルを追加しました。
* /etc —&amp;gt からの Image Server 設定の移行/conf.
* 翻訳用のソースコンテンツフラグメントを追加しました。
* ARIA のサポートを印刷および印刷ダイアログに追加しました。
* E メール検証 ARIA のサポートを追加しました。
* platform.clientlibs の修正に対する事前のバックポート。
* 明示的な dataType への入力がない場合の、スクリプトの自動実行の防止（CVE-2015-9251 を解決します）。

#### NPR-24536 に必要な設定 {#configuration-settings-required-for-npr-1}

共有キューの数は、デフォルトでは、ユーザーがタスクを要求したときに他のユーザーに対して更新されません。このため、新しいプロパティを導入しました。次の手順に従って、AEM インスタンスでこのプロパティを設定します。

1. 管理 UI／サービス／ワークスペース／グローバル管理に移動します。
1. グローバル設定を書き出します。
1. ダウンロードした XML ファイルで、タグを追加します。 &lt;client_taskspollinginterval>10&lt;/client_taskspollinginterval> ここでは、値の例を秒単位で示しています。 適宜変更できます。 
1.  ファイルを保存します。
1. 管理 UI／サービス／ワークスペース／グローバル管理に戻ります。
1. グローバル設定を読み込みセクションで xml ファイルを読み込みます。
1. これで、システムからログアウトして、再びログインできます。
1. ワークスペース内の他の開始が更新する共有キューユーザーの数です。
1. ポーリングをオフにするには、値を 0 に変更し、XML ファイルを再度読み込みます。

AEM 6.4.8.0 の Uber Jar は、 [Adobeの公開 Maven リポジトリ](https://repo.adobe.com/nexus/content/groups/public/com/adobe/aem/uber-jar/6.4.8/).

```shell
<dependency>

      <artifactId>uber-jar</artifactId>
      <version>6.4.8</version>
      <classifier>apis</classifier>
      <scope>provided</scope>
</dependency>
```
