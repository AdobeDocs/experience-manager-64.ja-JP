## AEM 6.4.7.0 に含まれる機能 {#what-s-included-in-aem}

AEM 6.4.7.0 is an important update that includes performance, stability, security and key customer fixes and enhancements released since the general availability of AEM 6.4 in **April 2018.**

また、これは累積的な機能であり、6.4.7.0には、その前にリリースされたすべてのAEM 6.4サービスパックが含まれています。

AEM 6.4.7.0の主な特徴は次のとおりです。

* 組み込み型のリポジトリ（Apache Jackrabbit Oak）をバージョン 1.8.17 に更新しました。
* サイトページを削除する際のバージョン設定のサポートを追加しました。
* 作成日の新しい列（並べ替え可能）が、 **DAMリスト表示およびリスト表示のアセット検索結果に追加されました** ( **** NPR-31311)。
* 「名前」列に基づくアセ **ットの並べ替え** は、リストビューで許可さ **れています** 。
* 再処理とバッチアップロードのバッチサイズとワークフローステップのタイムアウトが、ダイナミックメディアのUIから設定できるようになりました。
* Scene 7の `pdfBrochure` クラウド設定では、IPSでメモリを保存するために、がfalseに設定されています。

#### AEM 6.4.5.0 {#experience-manager-6450}

AEM 6.4.5.0 is an important update that includes performance, stability, security and key customer fixes and enhancements released since the general availability of AEM 6.4 in **April 2018.**

#### AEM 6.4.4.0 {#experience-manager-6440}

AEM 6.4.4.0 is an important update that includes performance, stability, security and key customer fixes and enhancements released since the general availability of AEM 6.4 in **April 2018.**

また、これは累積的な機能であり、6.4.4.0には、その前にリリースされたすべてのAEM 6.4サービスパックが含まれています。

AEM 6.4.4.0の主な特徴は次のとおりです。

* 組み込み型のリポジトリ（Apache Jackrabbit Oak）をバージョン 1.8.11 に更新しました。
* サービスバージョンHTTP要求が頻繁に発生するのを防ぐために、サービスバージョンのキャッシュのサポートが追加されました。
* 自動タグの削除のサポートを追加しました。
* カタログウィザードの無限スクロールを実装。
* コミュニティサイトに従って権限を制限するバックポート機能。
* Sling Granite Content Health checkのパフォーマンスの修正（キャッシュ、非同期実行、除外リスト）。
* ページサムネールダイアログにassetpickerボタンが追加されました。
* フィールド上でのツールチップの位置を許可する新しいプロパティが追加されました。
* マルチフィールド内のColorPickerのサポートを改善。
* コンテンツフラグメントのclientlibsにある数値入力マルチフィールドの空の値を無視するチェック機能を追加しました。
* Microsoft Translator Text API v3のサポートを有効にしました。

#### AEM 6.4.3.0 {#experience-manager-6430}

AEM 6.4.3.0は、パフォーマンス、安定性、セキュリティ、お客様向けの主要な修正に加え、2018年4月のAEM 6.4の一般リリース(GA)以降にリリースされた機能強化を含む重要なアップデートです。

* AEMでのパネルベースのアセット検索の強化
* ログインページのクロスサイトスクリプティング(XSS)に関する修正。
* UI計測が改善されました。
* FormDataの処理が改善されました。
* 複数フィールド内での項目名の処理を改善。
* 選択中のプレースホルダー項目（カード表示およびリスト表示）の処理を改善。
* マネージドサービスのAdobe IMS認証と管理コンソールのサポートを追加しました。

#### AEM 6.4.2.0 {#experience-manager-6420}

AEM 6.4.2.0 is an important update that includes performance, stability, security and key customer fixes and enhancements released since the general availability of AEM 6.4 in **April 2018.**
また、これは累積的な機能であり、6.4.2.0には、その前にリリースされたすべてのAEM 6.4サービスパックが含まれています。

AEM 6.4.2.0の主な特徴は次のとおりです。

* 組み込み型のリポジトリ（Apache Jackrabbit Oak）をバージョン 1.8.7 に更新しました。
* HTML Template Language(HTL)仕様1.4のサポートを追加しました。
* MongoDB Enterprise 3.6のサポートを追加しました。
* サイトページエディターでは、 <a href="../sites-developing/spa-walkthrough.md">AEMのSPA Editor JS SDKと組み合わせて、ReactまたはAngularで構築されたクライアント側コンポーネントを使用したコンテキスト内編集と組版のサポートが追加されます</a>。
* コンテンツフラグメントの機能強化：テキストフィールドに注釈を付ける機能と、バージョンを並べて比較する機能が追加されました。
* Adobe stockと [の統合が追加され、](/help/assets/aem-assets-adobe-stock.md) AEMユーザーインターフェイスから直接Adobe stockアセットを検索、プレビュー、保存、ライセンス認証できるようになりました。 詳しくは、「AEM AssetsでのAdobe Stockア [セットの使用」を参照してください](https://helpx.adobe.com/experience-manager/kt/assets/stock-assets-feature-video-use.md)。
* アセットに、動的な条件付きメタスキーマのサポートと、アセットフォルダーのメタデータスキーマの設定機能が追加されました。
* 各コンポーネントに、フォルダーサムネールの作成/更新機能を有効/無効にする設定を追加しました。
* ページオーサリングでの画像エディターの機能強化。
* 選択した回答が削除された場合に、スコアイベントのCommunitiesの回帰修正が正しく処理されませんでした。
* solrの検索結果の最大数をMAX_INT-10000に変更しました。
* トランザクションフラッシュジョブは、storeTransactionへの最初の呼び出し時にのみ開始されます。
* 「すべて選択」ボタンが、カード表示と列表示で使用できるようになりました。
* CoralUIのアクセシビリティが改善されました。
* Coral.Keys処理を改善。
* moment.jsを2.22.2に更新しました。
* jquery 1.12.1に更新

AEM 6.4.1.0は、パフォーマンス、安定性、セキュリティ、お客様向けの主要な修正に加え、2018年4月のAEM 6.4の一般リリース(GA)以降にリリースされた機能強化を含む重要なアップデートです。

AEM 6.4.1.0はAEM 6.4 GAにインストールできます。 サービスパックの主な特徴は次のとおりです。

* 組み込み型のリポジトリ（Apache Jackrabbit Oak）をバージョン 1.8.3 に更新しました。
* 強化されたスマートタグが導入されました。
* デザイン選択、タグ選択の修正（ソースVMとターゲットVMを自動に変更）
* 値を文字列として保存するためのtypeHintのサポートを追加しました。
* メールサービスでSMTP over TLSのサポートを追加しました。
* HTTPフォワーダーのプロキシ処理の修正(DMS7)。
* /etc/clientlibs/social/thirdparty/handlebars/source/handlebars.jsバージョン1.3に更新しました。
* DMHybridオプトアウトおよびオプトインパッケージの修正点です。
* スマート切り抜きの修正点です。
* OOTB構成値を新しい場所（/etcから/conf）に移行しました。
* 配信サーバーの顧客設定にカラープロファイルを追加しました。
* /etc —&amp;gt；からのImage Server設定の移行/conf.
* 翻訳用のソースコンテンツフラグメントを追加しました。
* PrintとPrintDialogにARIAサポートを追加しました。
* ARIAの電子メール検証サポートを追加しました。
* platform.clientlibsに対するプロアクティブなバックポートの修正。

#### NPR-21268 に必要な設定 {#configuration-settings-required-for-npr}

Classic（古い）UIを使用し、それを使用してメタデータテンプレートを作成した場合は、JMXスクリプトを実行してそれらを保存する手順に従います。

1. /system/console/jmxに移動します。
1. 「メタデータテンプレートの移行」をクリックします。
1. 「migrateMetadataTemplatesAtPath」をクリックし、メタデータテンプレートを保持するフォルダーのパスを指定します。

#### NPR-24536 に必要な設定 {#configuration-settings-required-for-npr-1}

共有キューの数は、デフォルトでは、ユーザーがタスクを要求したときに他のユーザーに対して更新されません。 このために、新しいプロパティを導入しました。 次の手順に従って、AEMインスタンスでこのプロパティを設定します。

1. 管理UI/サービス/Workspace/グローバル管理に移動します。
1. グローバル設定の書き出し
1. ダウンロードしたXMLファイルに、&lt;client_tasksPollingInterval>10&lt;/client_tasksPollingInterval>タグを追加します。ここでは、10が秒単位のサンプル値です。 それに応じて変更できます。
1. ファイルを保存します。
1. 管理UI/サービス/Workspace/グローバル管理に戻ります。
1. 「グローバル設定を読み込み」セクションでxmlファイルを読み込みます。
1. これで、システムからログアウトして、再度ログインできます。
1. 共有キューのカウントは、ワークスペース内の他のユーザーに対して更新を開始します。
1. ポーリングをオフにするには、値を0に変更し、XMLファイルを再度読み込みます。
